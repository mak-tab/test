# Документация по API "Библиотека"

### Это руководство поможет вам запустить, настроить и протестировать API для управления книгами и пользователями.

## Начало работы
1. Установите зависимости:
    ```
    pip install -r requirements.txt:
    ```

2.  Создайте файл `.env`:
    ```
    SECRET_KEY="09d25e094faa6ca2556c818166b7a9563b93f7099f6f0f4caa6cf63b88e8d3e7"
    ALGORITHM="HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES=30
    ```

3. Запуск сервера
    ```
    uvicorn main:app --reload
    ```
**Сервер будет доступен по адресу http://127.0.0.1:8000. При первом запуске он автоматически создаст файлы storage.json и users.json.**

## Аутентификация

### Шаг 1: Регистрация
- **POST** 
    `/reg`

- **Body (JSON):**
  ```json
    { 
        "username": "user1", 
        "password": "pass123" 
    }
  ```

- **Ответ:** 
    201, данные пользователя

- **Ошибки:** 
    400 — пользователь уже существует

### Шаг 2: Вход и получение токена
- **POST** 
    `/login`

- **Body (JSON):**
  ```json
    { 
        "username": "user1", 
        "password": "pass123" 
    }
  ```

- **Ответ:** 
    200, `{ "access_token": "...", "token_type": "bearer" }`

- **Ошибки:** 
    401 — неверные данные

- **Лимит:** 
    5 попыток в минуту

### Шаг 3: Использование токена в Postman
*Для всех защищенных запросов:*
*Перейдите на вкладку Authorization.*
*Выберите Type: Bearer Token.*
*Вставьте скопированный access_token в поле Token.*

## Эндпоинты
### Пользователи 
#### GET /users/me
Получение информации о текущем аутентифицированном пользователе.

Аутентификация: Требуется

Успешный ответ (200 OK):
```json
{
    "username": "myuser"
}
```

#### DELETE /users/me
Удаление аккаунта текущего пользователя.

Аутентификация: Требуется

Успешный ответ: 204 No Content (тело ответа пустое).

### Книги
#### GET /books
Получение списка всех книг.

Аутентификация: Не требуется

Успешный ответ (200 OK): Массив объектов книг.

#### GET /books/{book_id}
Получение информации о книге по её id.

Аутентификация: Не требуется

URL: http://127.0.0.1:8000/books/1

Успешный ответ (200 OK): Объект книги.

Ошибка (404 Not Found): Если книга с таким id не найдена.

#### POST /books
Добавление новой книги.

Аутентификация: Требуется

Тело запроса (Body → raw → JSON):

```json
{
    "title": "1984",
    "author": "George Orwell"
}
```
Успешный ответ (201 Created): Объект созданной книги с присвоенным id.

#### PUT /books/{book_id}
Полное обновление информации о книге по её id.

Аутентификация: Требуется

URL: http://127.0.0.1:8000/books/1

Тело запроса (Body → raw → JSON):

```json
{
    "title": "Nineteen Eighty-Four",
    "author": "George Orwell"
}
```
Успешный ответ (200 OK): Объект обновленной книги.

Ошибка (404 Not Found): Если книга с таким id не найдена.

#### DELETE /books/{book_id}

Удаление книги по её id.

Аутентификация: Требуется

URL: http://127.0.0.1:8000/books/1

Успешный ответ: 204 No Content (тело ответа пустое).

Ошибка (404 Not Found): Если книга с таким id не найдена.

## Коды ошибок
200 OK: Запрос успешно выполнен.

201 Created: Ресурс успешно создан.

204 No Content: Запрос успешно выполнен, но тело ответа отсутствует (используется для DELETE).

400 Bad Request: Ошибка в запросе. Например, попытка регистрации с уже существующим username.

401 Unauthorized: Ошибка аутентификации. Токен отсутствует, недействителен или истек срок его действия.

404 Not Found: Запрашиваемый ресурс (например, книга) не найден.

422 Unprocessable Entity: Ошибка валидации данных. Например, слишком короткий пароль или незаполненное поле title. Тело ответа будет содержать подробную информацию об ошибке.

429 Too Many Requests: Слишком много запросов за короткий промежуток времени (защита от brute-force).
