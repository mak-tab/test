# Документация Library API

## Описание

REST API для управления пользователями и книгами на FastAPI с хранением данных в JSON-файлах. Поддерживает регистрацию, аутентификацию (JWT), CRUD для книг, ограничение по токенам пользователя и лимитирование запросов. Удобно тестируется через Postman.

## Быстрый старт

1. Установите зависимости:
    ```
    pip install -r requirements.txt
    ```

2. Создайте файл `.env` в папке `api_json` и добавьте:
    ```
    KEY=your_secret_key
    ALGORITHM=HS256
    TOKEN_MINUTE=30
    ```

3. Запустите сервер:
    ```
    uvicorn main:app --reload --host 0.0.0.0 --port 8000
    ```

### 1. Регистрация пользователя
- **POST** 
    `/reg`

- **Body (JSON):**
  ```json
  { "username": "user1", "password": "pass123" }
  ```

- **Ответ:** 
    201, данные пользователя

- **Ошибки:** 
    400 — пользователь уже существует

### 2. Получение токена (логин)
- **POST** 
    `/token`

- **Body (JSON):**
  ```json
  { "username": "user1", "password": "pass123" }
  ```

- **Ответ:** 
    200, `{ "access_token": "...", "token_type": "bearer" }`

- **Ошибки:** 
    401 — неверные данные

- **Лимит:** 
    5 попыток в минуту

### 3. Получение информации о себе
- **GET** 
    `/users/me`

- **Headers:** 
    `Authorization: Bearer <access_token>`

- **Ответ:** 
    200, данные пользователя

- **Ошибки:** 
    401 — невалидный токен

### 4. CRUD для книг

#### Добавить книгу
- **POST** 
    `/books/`

- **Headers:** 
    `Authorization: Bearer <access_token>`

- **Body (JSON):**
  ```json
  { "title": "Book Title", "author": "Author Name" }
  ```

- **Ответ:** 
    201, данные книги

- **Ошибки:** 
    403 — закончились токены

#### Получить список книг
- **GET** 
    `/books/`

- **Headers:** 
    `Authorization: Bearer <access_token>`

- **Ответ:** 
    200, список книг

#### Обновить книгу
- **PUT** 
    `/books/{book_id}`

- **Headers:** 
    `Authorization: Bearer <access_token>`

- **Body (JSON):**
  ```json
  { "title": "New Title", "author": "New Author" }
  ```

- **Ответ:** 
    200, обновлённая книга

- **Ошибки:** 
    404 — книга не найдена или нет прав

#### Удалить книгу
- **DELETE**
    `/books/{book_id}`

- **Headers:** 
    `Authorization: Bearer <access_token>`

- **Ответ:** 
    204 

- **Ошибки:** 
    404 — книга не найдена, 
    403 — нет прав

## Коды ошибок
- 400 — Некорректные данные
- 401 — Неавторизован/неверный токен
- 403 — Нет прав или закончились токены
- 404 — Не найдено
- 429 — Слишком много запросов

## Примечания
- Каждый запрос к защищённым эндпоинтам уменьшает количество токенов пользователя (`tokens_left`).
- Для тестирования используйте коллекцию Postman, добавляя токен авторизации в заголовки.
- Все данные хранятся в `users.json` и `books.json`.
